Week 16 (May 20th - May 24th)

During this sixteenth week, I focused on addressing the issue that I couldn't solve the previous week, which involved identifying the exercise URL when deploying the smart tutor container. This would enable the execution of a different question-answer model based on the exercise the client was on. After numerous attempts, I managed to fix the problem and subsequently submitted a Pull Request with all the changes to the smart tutor, ensuring that not only I could access this tutor.

Additionally, I submitted a Pull Request with all the changes to the unibotics-webserver to ensure the tutor could function correctly.

After fixing the problem, I started working on optimizing the RADI, researching and exploring ways to optimize it. However, I encountered an issue when my computer malfunctioned, preventing it from utilizing the GPU at all times. As a result, I couldn't conduct optimization tests since my computer only utilizes the CPU.

Unable to progress on that front, I shifted my focus to the visual aspect of the tutor, making it more appealing to the client by incorporating the characteristic colors of the page, font changes, etc.



Finally, as with every week, I updated the blog with another week's worth of progress.
